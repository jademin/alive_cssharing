# 기획 방향 선택 규칙

## 목적
사용자가 콘텐츠 유형을 고르지 않는다. AI가 입력 초안에서 서로 다른 기획 방향 3안을 제안하고, 사용자가 하나를 고른다.

## 중요: 초안이 이미 제공된 경우
사용자 메시지에 `[작성자 초안]` 섹션이 포함되어 있으면, 방향 선택이 이미 완료된 것이다.
그 초안의 중심 질문과 핵심 주장을 선택된 방향으로 간주하고, 방향 제안 없이 즉시 JSON을 생성한다.

## 방향의 기준
- 기획 방향은 상황 해부형, 판단 분기형 같은 형식 이름이 아니라 이번 콘텐츠가 답할 중심 질문과 핵심 주장이다.
- 방향 A, B, C는 문장만 바꾸지 않는다. 독자가 새롭게 이해할 운영 관점이 달라야 한다.
- 아래 관점 중 입력 초안에 가장 맞는 것을 활용한다.
  - 고객이 구매·사용·주문 이후 어느 시점에서 멈추는가
  - 반복 문의가 어떤 안내 또는 운영 단절을 보여주는가
  - 상담사가 즉시 판단하지 못하는 이유는 무엇인가
  - 서로 다른 정보 출처나 담당 영역이 어디에서 연결되지 않는가
  - 출시 또는 운영 전에 무엇을 점검해야 하는가

## 방향으로 쓰지 않는 결론
다음 문장은 방향의 핵심 주장으로 쓰지 않는다.
- 문의 유형별 확인 정보가 다릅니다
- 체계적인 운영이 필요합니다
- 응대 기준을 세워야 합니다
- 문의를 분류해야 합니다
- 운영 효율을 높여야 합니다

## story_plan 작성 규칙
- story_plan은 Instagram / Facebook의 중간 카드 계획이다.
- 3개에서 4개로 작성한다. 첫 장과 CTA 장은 story_plan에 넣지 않는다.
- story_plan이 3개면 총 5장, 4개면 총 6장으로 구성한다.
- 각 단계는 한 카드의 다른 메시지 또는 근거를 담당한다.
- 여러 상세 사례가 하나의 상위 주장에 대한 병렬 사례라면, 한 카드 items로 묶는 방향을 우선한다.
- 하나의 사례를 원인, 확인 정보, 응대 방식으로 쪼개 여러 카드에 반복하지 않는다.
- story_plan은 3개 미만으로 줄이지 않는다. 카드 수를 맞추기 위해 일반적인 운영 절차를 추가하지 않는다.

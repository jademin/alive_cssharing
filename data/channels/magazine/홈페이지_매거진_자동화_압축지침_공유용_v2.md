# 홈페이지 매거진 자동화 압축 지침 v2

## 1. 문서 목적

이 문서는 홈페이지 매거진 콘텐츠를 자동화 프로그램에서 생성할 때 적용하기 위한 공통 작성 기준이다.

자동화 프로그램은 매거진 원고와 기본 구성을 빠르게 생성하는 데 사용한다. 다만 자동화 결과물은 최종 발행본이 아니라 1차 생성본으로 본다. 실제 제출용 산출물은 생성 결과를 바탕으로 이미지, CTA, HTML 화면, 검수용 정보 비노출 여부를 함께 확인한 뒤 정리한다.

이 문서는 특정 아티클 전용 지침이 아니라, 홈페이지 매거진 작업 전반에 반복 적용할 수 있는 공통 기준이다.

---

## 2. 기본 역할

AI는 홈페이지 매거진 콘텐츠 작성 보조자로 작동한다.

수행 역할은 다음과 같다.

1. 주제에 맞는 매거진 원고 작성
2. 제목, 메타 설명, 키 프레이즈 정리
3. CTA 전 문단 작성
4. FAQ 작성
5. 썸네일 및 CTA 이미지 반영 위치 확인
6. HTML 미리보기 구조 정리
7. 제출 전 검수 항목 확인

---

## 3. 매거진 콘텐츠의 목적

홈페이지 매거진은 잠재 고객이 CS쉐어링의 고객센터 운영 전문성을 이해하고, 필요 시 상담으로 자연스럽게 이동할 수 있도록 돕는 콘텐츠다.

콘텐츠는 단순 홍보문이 아니라 고객센터 운영 상황, 문제 지점, 점검 기준, 해결 방향을 실무적으로 설명하는 글이어야 한다.

독자가 본문을 읽고 우리 회사 상황에 어떤 고객센터 운영 기준이 필요한지 이해할 수 있어야 한다.

## 4. 최종 산출물 기준

매거진 1건의 산출물은 아래 요소를 포함한다.

| 항목 | 기준 |
|---|---|
| 제목 | 키 프레이즈를 자연스럽게 포함 |
| 메타 설명 | 90~140자 내외 |
| 키 프레이즈 | AIOSEO 입력용 핵심 문구 1개 |
| 본문 | 도입, 문제 지점, 점검 기준, 운영 방향, CTA 흐름 |
| FAQ | 2~4개 |
| CTA 전 문단 | 본문 주제와 상담 필요 기준을 자연스럽게 연결 |
| CTA 버튼 문구 | 지금 바로 상담받기 |
| 썸네일 | 대표 이미지 영역에 반영 |
| CTA 보조 이미지 | 버튼형 CTA를 보조하는 작은 이미지로 반영 |
| 검수용 정보 | 실제 HTML 화면에는 노출하지 않음 |

## 5. 본문 작성 기준

본문은 아래 흐름을 기본으로 한다.

```text
주제 상황 제시
→ 고객센터 운영상 문제 또는 확인 필요 지점
→ 핵심 점검 기준
→ 운영 방향 정리
→ FAQ
→ CTA
```

콘텐츠 유형에 따라 일부 순서는 조정할 수 있다.

### 문체 기준

- 실무자가 이해하기 쉬운 문장으로 작성한다.
- 과장된 광고 표현을 사용하지 않는다.
- 한 문장에는 하나의 메시지만 담는다.
- 같은 표현을 제목, 소제목, 본문에서 반복하지 않는다.
- 문제를 지적할 때는 불안감을 과도하게 키우지 않고 점검 기준으로 연결한다.
- CS쉐어링은 문제를 단정적으로 해결한다고 표현하지 않고, 기업 상황에 맞는 운영 방향을 함께 검토하는 방식으로 표현한다.

---

## 6. 제목 작성 기준

제목은 검색성과 클릭 가능성을 모두 고려한다.

### 기준

- 키 프레이즈를 자연스럽게 포함한다.
- 너무 광고성으로 보이지 않게 작성한다.
- 고객센터 운영 상황이나 문제 지점이 드러나게 작성한다.
- 제목 안에서 같은 단어를 반복하지 않는다.
- 한글 단어가 중간에서 끊기지 않도록 줄바꿈을 고려한다.

### 피해야 할 제목

- 과장된 성과를 보장하는 제목
- “무조건”, “완벽”, “최고”, “1위” 등 검증 어려운 표현
- 본문에서 다루지 않는 내용을 암시하는 제목
- 키워드를 억지로 반복한 제목

---

## 7. 메타 설명 작성 기준

메타 설명은 검색 결과에서 글의 내용을 이해할 수 있도록 작성한다.

### 기준

- 90~140자 내외로 작성한다.
- 키 프레이즈를 자연스럽게 포함한다.
- 본문 핵심 내용을 요약한다.
- 상담 유도보다 정보 확인 목적을 우선한다.
- 클릭을 과도하게 유도하는 표현은 피한다.

---

## 8. 키 프레이즈 기준

키 프레이즈는 AIOSEO 입력용 핵심 문구 1개만 지정한다.

### 기준

- 본문 주제와 직접 연결되는 문구를 선택한다.
- 너무 넓은 단어보다 검색 의도가 분명한 문구를 우선한다.
- 제목과 메타 설명에 자연스럽게 포함한다.
- 본문에서는 억지로 반복하지 않는다.

---

## 9. CTA 작성 기준

CTA는 본문 내용을 상담 행동으로 자연스럽게 연결하는 역할을 한다.

CTA는 버튼형을 기본으로 한다.

### 기본 구조

```text
CTA 전 문단
+
지금 바로 상담받기 버튼
+
작은 보조 이미지
```

### CTA 전 문단 기준

CTA 전 문단은 본문에서 다룬 고객센터 운영 문제 또는 확인 필요 지점을 기준 점검으로 연결한다.

예시 구조:

```text
[본문에서 다룬 운영 문제]가 반복된다면, 먼저 [상담 유형·운영 기준·품질관리 기준]을 점검할 필요가 있습니다. CS쉐어링은 기업 상황에 맞는 고객센터 운영 방향을 함께 검토합니다.
```

### CTA 버튼 기준

- 버튼 문구는 `지금 바로 상담받기`를 사용한다.
- CTA URL은 버튼 링크에만 적용한다.
- CTA URL 원문은 화면 텍스트로 직접 노출하지 않는다.

### 금지 표현

- 무료 상담
- 무조건 해결
- 완벽한 해결
- 업계 최고
- 지금 바로 클릭하세요
- 상담만 받으면 해결됩니다
- 모든 고객센터 문제를 해결합니다

---

## 10. 썸네일 기준

썸네일은 본문 중간 이미지가 아니라 매거진 대표 이미지 또는 로컬 미리보기용 대표 이미지 영역에 적용한다.

### 기준

- 공식 마스코트 1개를 사용한다.
- 주제와 연결되는 소품 1~2개만 사용한다.
- 충분한 여백을 둔다.
- 이미지 안에 제목, 요약문, 체크리스트 문구를 넣지 않는다.
- 정보 카드처럼 보이지 않게 한다.
- 공식 마스코트의 형태, 색상, 비율, 얼굴 구조가 유지되어야 한다.
- 최종 검수 전 이미지는 `임시 연결용` 또는 `검수 대기` 상태로 관리한다.

---

## 11. CTA 보조 이미지 기준

CTA 보조 이미지는 CTA 전 문단과 버튼을 보조하는 작은 시각 요소다.

### 기준

- 버튼보다 작게 배치한다.
- CTA 메시지를 대신하지 않는다.
- 소품은 1개 수준으로 제한한다.
- 상담 말풍선, 체크리스트, 알림 아이콘, 상담 아이콘 등을 사용할 수 있다.
- 마스코트가 CTA 버튼보다 먼저 보이거나 더 강조되면 조정한다.
- CTA 이미지가 없으면 CTA 비주얼 요소 누락으로 본다.

## 12. HTML 화면 기준

제출용 HTML은 실제 독자 화면처럼 보여야 한다.

### HTML에 포함할 요소

- 대표 이미지 또는 썸네일
- 제목
- 서브헤드
- 본문
- 표 또는 핵심 항목
- FAQ
- CTA 전 문단
- CTA 버튼
- CTA 보조 이미지

### HTML에 노출하지 않을 요소

- 키 프레이즈
- 메타 설명
- 이미지 alt 설명
- 검수표
- 내부 입력값
- 프롬프트
- CTA URL 원문

## 13. HTML CSS 기준

한글 단어가 중간에서 끊기지 않도록 아래 기준을 반영한다.

```css
h1, h2, h3, p, li, td, th,
.subhead, .summary, .card, .faq, .cta {
  word-break: keep-all;
  overflow-wrap: break-word;
}
```

제목이 길면 의미 단위로 줄바꿈한다.

## 14. FAQ 기준

FAQ는 독자가 실제로 궁금해할 질문 중심으로 작성한다.

### 기준

- 2~4개로 제한한다.
- 본문 내용을 단순 반복하지 않는다.
- 상담 전 확인할 기준을 정리하는 방향으로 작성한다.
- 답변은 짧고 명확하게 쓴다.

---

## 15. 표와 체크리스트 기준

표 또는 체크리스트는 본문 이해를 돕는 핵심 요소로 사용한다.

### 기준

- 표는 1개 중심으로 사용한다.
- 표 안 문장은 짧게 작성한다.
- 동일한 내용을 본문, 표, FAQ에서 반복하지 않는다.
- 체크리스트는 실제 점검 가능한 항목으로 작성한다.

## 16. 이미지 파일 관리 기준

이미지 브리프와 실제 이미지 파일은 구분한다.

| 구분 | 설명 |
|---|---|
| 이미지 브리프 | 이미지 제작 지시서 |
| 실제 이미지 파일 | HTML에 연결되는 PNG/JPG/WebP/SVG 파일 |
| image_preview.html | 이미지 단독 확인용 보조 화면 |
| 제출용 HTML | 실제 매거진 화면 확인용 |

브리프만 있는 상태는 최종 제출 상태가 아니다.

최종 제출 전에는 실제 이미지 파일이 있고, HTML에서 이미지가 보여야 한다.

## 17. 이미지 파일명 기준

이미지 파일명은 위치, 주제, 마스코트를 식별할 수 있게 작성한다.

### 기본 규칙

```text
thumbnail_[article_slug]_[mascot].png
cta_[article_slug]_[mascot].png
```

### 예시

```text
thumbnail_peak_season_cs_charring.png
cta_peak_season_cs_charring.png
```

예시는 참고용이며, 실제 파일명은 아티클 주제와 이미지 상태에 맞게 조정할 수 있다.

## 18. 공용 asset 관리 기준

매거진 작업에서 사용하는 썸네일, CTA 보조 이미지, 마스코트 이미지, 이미지 브리프, HTML 연결 상태, 검수 결과는 공용 asset 관리 노트에서 관리할 수 있다.

공용 asset 관리 노트는 특정 아티클 전용 문서가 아니라 여러 매거진 산출물에 반복 적용할 수 있는 기준 문서다.

### 공용 asset 관리 노트에 포함할 항목

| 항목 | 기준 |
|---|---|
| 아티클명 | 개별 매거진 주제명 |
| article_slug | 파일명과 HTML 경로에 사용할 영문 식별자 |
| 사용 마스코트 | 쉐어러 / 시스 / 고미 베어 / 채링이 |
| 썸네일 파일 | 실제 대표 이미지 파일명 |
| CTA 파일 | 실제 CTA 보조 이미지 파일명 |
| 썸네일 상태 | 초안 / 임시 연결 / 검수 대기 / PASS / WARN / FAIL / 보류 |
| CTA 상태 | 초안 / 임시 연결 / 검수 대기 / PASS / WARN / FAIL / 보류 |
| HTML 연결 상태 | 이미지가 실제 화면에 표시되는지 여부 |
| 최종 판정 | 제출 가능 / 보류 / 재작업 |
| 남은 확인 사항 | 출처, 원형, 색상, 경로, 화면 노출 여부 등 |

## 19. 상태값 기준

이미지와 HTML 반영 상태는 아래 기준으로 관리한다.

| 상태 | 의미 |
|---|---|
| 초안 | 브리프 또는 후보 이미지만 있음 |
| 임시 연결 | HTML에 연결했지만 최종 검수 전 |
| 검수 대기 | 이미지와 HTML 연결은 있으나 사람 검수 전 |
| PASS | 제출 가능 |
| WARN | 제출 전 확인 필요 |
| FAIL | 재제작 또는 재연결 필요 |
| 보류 | 출처, 원형, 사용 가능 여부 확인 전 |

## 20. 자동화 생성본과 제출용 산출물의 구분

이 지침은 자동화 프로그램에 입력하기 위한 공통 작성 기준이다.

자동화 프로그램으로 생성한 결과물은 원고와 구성의 1차 생성본으로 본다. 실제 제출용 산출물은 자동화 생성본을 그대로 제출하는 것이 아니라, 제출 형식에 맞게 이미지, CTA, HTML 화면, 검수용 정보 비노출 여부를 함께 확인한 결과물이다.

따라서 자동화 생성본과 제출용 HTML 미리보기는 산출물 단계가 다르다.

제출용 HTML에서는 아래 항목을 추가로 확인한다.

| 항목 | 확인 기준 |
|---|---|
| 썸네일 | 대표 이미지 영역에 실제 표시되는지 확인 |
| CTA 이미지 | CTA 영역에서 버튼을 보조하는 위치에 표시되는지 확인 |
| CTA 버튼 | 버튼 문구와 링크가 적용되어 있는지 확인 |
| 화면 노출 정보 | 키 프레이즈, 메타 설명, 검수표, CTA URL 원문이 화면에 노출되지 않는지 확인 |
| 본문 흐름 | 자동 생성 문장이 실제 매거진 화면에서 자연스럽게 읽히는지 확인 |
| 파일 경로 | 이미지 파일 경로가 깨지지 않는지 확인 |

자동화 결과물은 최종본이 아니라 제출용 산출물로 정리하기 전 단계로 관리한다.

## 21. 검수 기준

생성 후 아래 항목을 확인한다.

| 검수 항목 | 기준 |
|---|---|
| 제목 | 키 프레이즈가 자연스럽게 포함됨 |
| 메타 설명 | 90~140자 내외 |
| 본문 흐름 | 문제 제기에서 운영 기준으로 자연스럽게 이어짐 |
| 반복 표현 | 제목, 소제목, 본문에서 같은 표현 과다 반복 없음 |
| FAQ | 2~4개 |
| CTA | 전 문단 + 버튼 + 보조 이미지 포함 |
| CTA URL | 화면 텍스트로 직접 노출되지 않음 |
| 썸네일 | 대표 이미지 영역에 표시됨 |
| CTA 보조 이미지 | 버튼형 CTA를 보조하는 위치에 표시됨 |
| 마스코트 | 공식 캐릭터로 식별 가능 |
| 내부 정보 | 검수표, 프롬프트, 메타 설명 등이 HTML 화면에 노출되지 않음 |

---

## 22. 실패 기준

아래 중 하나라도 해당하면 수정 또는 재작업한다.

- 실제 이미지 파일이 없다.
- HTML에서 이미지가 보이지 않는다.
- placeholder 문구가 남아 있다.
- 썸네일이 정보 카드처럼 보인다.
- CTA에 전 문단과 버튼만 있고 보조 이미지가 없다.
- CTA URL 원문이 화면에 직접 노출된다.
- 공식 마스코트와 다른 캐릭터처럼 보인다.
- 키 프레이즈, 메타 설명, 검수표가 HTML 화면에 노출된다.
- 본문에서 같은 표현이 반복되어 읽기 흐름이 떨어진다.

---

## 23. 출력 형식

매거진 콘텐츠를 생성할 때는 아래 순서로 출력한다.

```text
1. 제목
2. 메타 설명
3. 키 프레이즈
4. 서브헤드
5. 본문
6. FAQ
7. CTA 전 문단
8. CTA 버튼 문구
9. 썸네일 이미지 파일명
10. CTA 이미지 파일명
11. HTML 반영 시 주의사항
12. 검수 체크리스트
```

단, 실제 제출용 HTML 화면에는 2번, 3번, 9번, 10번, 11번, 12번을 화면 텍스트로 노출하지 않는다.

해당 항목은 검수용 문서 또는 내부 입력값으로만 관리한다.

---

## 24. 최종 원칙

매거진은 정보 제공형 콘텐츠다.

CTA는 본문을 대체하지 않고, 본문에서 정리한 확인 지점을 상담 행동으로 연결하는 역할만 한다.

썸네일과 CTA 이미지는 장식이 아니라 브랜드 인식과 콘텐츠 이해를 돕는 보조 요소다.

원고, 이미지, HTML, 검수 문서는 서로 역할을 분리해서 관리한다.

# 08. 브랜드·CTA 레퍼런스

> CTA에 사용할 실제 연락처·URL과 "어떤 솔루션을 연결?" 판단을 위한 솔루션 카탈로그를 담습니다.
> researcher / writer / image-maker 에이전트가 참조합니다.
> 실제 값이 바뀌면 이 파일만 업데이트하면 전체 에이전트에 반영됩니다.

## 1. CTA 연락처
| 항목 | 값 |
|---|---|
| 대표전화 | **1522-5539** |
| 상담시간 | 월요일 09:00 ~ 18:00 (주말·공휴일 제외) |
| 기본 URL | **https://cssharing.com/** |

### 블로그 CTA URL 패턴

```
https://cssharing.com/?utm_source=naver_blog&utm_medium=organic&utm_campaign=[주제-슬러그]
```

- `[주제-슬러그]`는 각 주제를 영문/숫자 슬러그로 변환(예: AICC 관련 글 → `aicc`).
- **네이버 블로그 자체 추적기(bsa)는 UTM을 덮어씌우지 않는다.** 애널리틱스 성과 데이터와 블로그 유입 데이터가 따로 존재하더라도 애널리틱스가 성과 판단에 우선한다.

## 3. 브랜드 이미지

| 항목 | 값 |
|---|---|
| 위치 | `assets/brand/` (프로젝트 루트, **주제와 무관한 고정 디렉터리** 유지 — `output/[주제]/`와 구분) |
| `cs쉐어링-로고.png` | 브랜드 로고마스터 (표준 가로형) — 첨부나 설명 문서의 `<img>`로 직접 활용 |
| `cs쉐어링-마스코트.png` | **메인 이미지(썸네일) 합성 전용** — 이 이미지를 그냥 복사해서 쓰는 작업자는 반드시 읽는다. 사용법은 `04-image-guide.md` 1절 참고 |

**새 자산 추가**: 새 브랜드 이미지가 생기면 `assets/brand/` 폴더에 파일을 추가하면 된다. image-maker 에이전트는 매 작업 시작 시 이 폴더를 스캔해 폴더에 있는 모든 이미지를 브랜드 자산으로 자동 인식한다. 파일명에 코드를 별도로 지정할 필요가 없다.

**파일명 규칙 예시**: 파일명에 "마스코트"가 포함된 이미지는 메인 이미지 합성용으로, "로고"가 포함된 이미지는 로고 마스터로 자동 인식된다. 지금 없으면 `04-image-guide.md`의 기본 스타일로 브랜드 자산 없이 생성한다.

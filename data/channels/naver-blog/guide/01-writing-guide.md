# 글쓰기 룰 (CS쉐어링 B2B)

**목적** — 독자(CS팀장·운영팀장·대표)가 "CS쉐어링을 써봐야겠다"는 생각이 들어야 한다.
소제목 4~6개 각각을 "문제 → 자사 서비스 해결"로 끝낸다. CTA 2곳에만 서비스를 몰지 않는다.

---

## 수치

| 항목 | 기준 |
|---|---|
| 제목 | 22~28자 (공백 포함, **직접 세기**) |
| 본문 | 2,000~2,500자 (공백 제외), 소제목 4~6개 |
| 이미지 | 6~8장, 캡션 1줄 필수 |
| 이모지 | 5~8개 |
| 메인 키워드 | 제목 맨 앞 + 본문 5~8회 (10회+ = 스터핑) |
| 서브 키워드 | 2~3개, 각 2~4회 |
| 해시태그 | 3~5개 |
| CTA | 2곳 (중간·마지막) |

---

## 구조

① 도입(250~350자) → ② 목차 `📑 이 글의 순서` → ③ 본문(1,200~1,500자) → ④ CTA 중간(100~150자) → ⑤ CTA 마지막(200~300자)

**도입 훅 패턴** (1개 선택): 공감형 🤔 / 질문형 💭 / 장면형 🎯 / 수치형 💡

**시그니처**: "A가 아니라 B" 대조 구문 1~2개

## 소제목·목차

- 이모지(0~1개) + 짧은 명사구. 승인 이모지: 🔍 📊 ⚡ 💬 📞 📦 🚚 🎯 📈
- `✅`·`-` 로 시작하는 행은 소제목 금지 (assembler가 리스트로 처리)
- 목차 항목 3~5개, 본문 소제목과 단어까지 정확히 일치

---

## 문장·단락

- 한 문장 = 한 줄 / 단락 2~4문장 / 모바일 3~4줄
- 단락 시작 패턴 다양화: 명사구·질문·장면·짧은 단정 혼용 (명사구 80% 이하)
- 접속사(또한·더불어·뿐만 아니라) 한 단락 2회+ 금지
- "첫째·둘째·셋째" 정형 병렬 금지

금지: 😂 😍 🥳 🎉 🔥 💪 (SNS·감정), 같은 이모지 3회+ 반복, 한 단락 2개+
